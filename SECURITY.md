# Security Policy

## Supported Versions

Only the latest `main` branch version of **AirHydra** is actively supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark:  |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of our users and developers seriously. If you discover a security vulnerability within AirHydra, please do **not** disclose it publicly via GitHub Issues.

### Reporting Process
1. **Private Disclosure**: Please report all vulnerabilities by reaching out to the maintainer directly through their GitHub profile or a private communication channel.
2. **Details**: Provide a detailed description of the flaw, including steps to reproduce the issue and a proof-of-concept (PoC) if possible.
3. **Response**: We aim to acknowledge all reports within 48 to 72 hours.
4. **Patching**: Once verified, a patch will be developed and released in the next update.

## Guidelines
- Avoid non-ethical research.
- Do not perform any testing against third-party networks or devices without explicit permission.
- Please provide us with a reasonable amount of time to resolve the issue before any public disclosure.
